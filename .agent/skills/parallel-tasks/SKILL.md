# Parallel Task Execution Skill

## 핵심 원칙

> [!IMPORTANT]
> **다중 요청 시 일괄 커밋 필수**
> 사용자가 여러 개를 요청하면 **모든 수정 완료 후** 한 번에 빌드/커밋/푸시

## 병렬 실행 조건

### 병렬 가능한 작업
- 서로 다른 파일 수정
- 동일 파일의 서로 다른 함수/컴포넌트 수정
- 독립적인 테스트 실행

### 순차 실행 필요한 작업
- 같은 파일의 같은 위치 수정
- 타입 정의 → 해당 타입 사용
- DB 마이그레이션 → 관련 서비스 코드
- 부모 컴포넌트 → 자식 컴포넌트

## 작업 분석 프로세스

```
[다중 요청 수신]
      │
      ▼
[의존성 분석]
      │
      ├── 독립적? ──> 병렬 실행
      │
      └── 의존적? ──> 순서 결정 후 순차 실행
      │
      ▼
[전체 빌드 검증]
      │
      ▼
[단일 커밋]
```

## 병렬 실행 패턴

### 예시: 3개 기능 동시 개발

사용자 요청: "히어로 애니메이션, 이미지 모달, 팀 탈퇴 기능 구현해줘"

병렬 수정 가능:
1. `LandingPage.tsx` (히어로)
2. `ImageUploadModal.tsx` (새 파일)
3. `TeamSettingsPage.tsx` (탈퇴)

→ 3개 파일 동시에 수정 후 한 번에 빌드

## 빌드 및 커밋 전략

### 병렬 수정 후 검증
```bash
# 빌드 검증
npm run build

# 타입 검증
npx tsc --noEmit

# 모든 변경사항 커밋
git add -A && git commit -m "feat: implement multiple features

#1 Feature A
#2 Feature B
#3 Feature C"

git push origin main
```

## 체크리스트

### 병렬 실행 전
- [ ] 작업 간 의존성 확인
- [ ] 파일 충돌 가능성 점검
- [ ] 타입 정의 우선순위 확인

### 병렬 실행 후
- [ ] 전체 빌드 성공
- [ ] TypeScript 에러 없음
- [ ] 기존 테스트 통과
- [ ] 단일 커밋 메시지 작성

## 주의사항

1. **같은 파일 다중 수정 금지**: 같은 파일을 여러 번 수정하면 마지막 수정만 적용됨
2. **타입 의존성**: 새 타입 → 사용하는 코드 순서로 작성
3. **import 경로**: 새 파일 생성 시 다른 파일의 import 경로 확인
4. **커밋 메시지**: 여러 기능을 명확히 구분하여 기록
